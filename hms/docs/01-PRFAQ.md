### 正文

2026 年的 LLM 用户终于面对一个新问题：**模型变多了，切换变贵了**。

一个典型的 agent 重度用户会同时挂 4 家以上 provider（火山方舟 / DeepSeek 官方 / Z.AI / OpenRouter），每家 6~10 个模型。早上要便宜快速，下午要 1M 上下文，晚上又要切到便宜模型对话——每一次切换都意味着手改 YAML、改 4 处字段、维护互斥、备份原文件、出错了找昨天的备份。

**单次 30 秒，但日均 5~10 次，出错率 10%，恢复成本 5 分钟。每天烧掉的注意力远比你想象的多。**

`hms` 是为这个具体问题设计的命令行工具。380 行 bash，零额外依赖，把"切换 provider"从一个"小工程"变成一个"日常动作"：

```
$ hms ds-pro
✓ 备份: config-20260622-001311.yaml
✓ 端点可达 (HTTP 401)
- provider: volcengine-agent-plan
+ provider: deepseek
✓ config.yaml 已原子更新
⚠  当前会话仍跑旧模型, /reset 或重启
   切错? → hms restore
```

## 2. FAQ（一个多疑用户会问的所有问题）

### Q1. 这不就是 `sed` + 备份吗？为什么要 380 行？
**A.** 单看切换动作确实是 sed。但要做到"切错了能撤、切之前知道端点活着、切的时候不会写一半崩了 YAML、切完之后能肉眼复核、互斥不会把没填 key 的占位误覆盖"——任何一个细节都需要十几行代码。380 行是这些细节的总和，不是过度工程。

### Q2. 为什么不做自动 fallback？当前 provider 限流时自动切走多方便。
**A.** 这是 v1 的红线设计决策。三个理由：
1. 模型决定输出质量，自动切换=偷偷换底层模型，调试时会怀疑人生
2. 不同 provider 流向不同计费账户，自动切=自动花别人的钱
3. 切错后人会知道"我刚才切错了"，自动切人不一定察觉

→ 工具职责限定在**探测、报告、提示**，不替人按按钮。

### Q3. 为什么不接 keychain / 1Password / Vault？
**A.** 信任本机文件系统 + chmod 600 已经够安全，引入系统级密钥管理会带来：
1. 跨设备解决方案复杂度（macOS keychain ≠ Linux secret-tool）
2. CI/无人值守场景被卡（脚本要解锁 keychain）
3. 调试时无法直接 cat 看 key 状态

简单到不会出错，是这个工具的核心约束。

### Q4. 为什么不用 Python / Go / Rust？
**A.** bash + 内嵌 python3 heredoc 是当前选项里 footprint 最小的组合。任何编译语言都需要分发二进制，任何"高级"脚本语言都需要 venv。这个工具的目标用户=自己，已知环境有 python3 自带 yaml，零分发成本。

### Q5. 380 行 bash 不会维护爆炸吗？
**A.** 不会，因为：
1. 边界明确：只读写 `model.*` 和 `providers.*.api_key`，不碰其他字段
2. 18 个预设硬编码在脚本里，要加新预设改源码即可（不引入外部配置层是有意的）
3. 单文件、零依赖意味着可以一眼读完

可维护性来自范围克制，不来自代码量。

### Q6. 切换之后当前 Hermes 会话能立刻生效吗？
**A.** 不能。`config.yaml` 在 Hermes 启动时加载，切换之后必须 `/reset` 或重启进程。这是 Hermes 框架的设计，不是 hms 的能力问题。文档明确写在 README 第一段。

### Q7. 我可以 `pip install hms` 用起来吗？
**A.** **不可以，这是 showcase 仓库，不是 product**。
它假设：
- 你装了 Hermes Agent
- 你的 `~/.hermes/config.yaml` 长成特定形状
- 你愿意按自己的偏好定义 provider 别名

直接 clone 跑不起来。要用的话 fork 改。

### Q8. 我可以贡献新预设吗？
**A.** 当前版本不接受外部 PR——它是个人工具的开源展示，不是社区项目。但欢迎 fork 你自己的版本。

### Q9. 为什么端点预检返回 HTTP 401 反而算"通过"？
**A.** 因为预检用的是假授权头（不浪费真 key 配额）。401 = DNS 通 + TLS 通 + API 网关在线 + 它认得你的请求格式（只是不认这个 key）。这是你能拿到的最便宜的"端点活着"信号。只有 HTTP code = 000（DNS 失败或超时）才算真不通。

### Q10. 金库被偷了怎么办？
**A.** 同任何 `~/.aws/credentials` 类的本地密钥文件一样：
1. 立即吊销所有 key
2. 重新生成
3. 用 `hms vault` 检查状态

工具不解决"主机被攻破"这一层威胁，假设本地 fs 是可信边界。
